# Remediation Ledger

This ledger records the remediation items implemented directly in the live repo, the current status, and any intentionally deferred work.

| Audit claim | Code status | Resolution |
| --- | --- | --- |
| Event bus backlog was unbounded and could amplify hot-path events. | Confirmed in prior `bot/core/event_bus.py`. | Replaced with a bounded bus that coalesces `KlineCloseEvent`, `BookTickerEvent`, `ShortlistUpdatedEvent`, and `OIRefreshDueEvent`, and exposes queue telemetry (`current_depth`, `high_water_mark`, `coalesced_count`, `dropped_count`). |
| `prepare_symbol()` and feature-store persistence ran synchronously on the event loop. | Confirmed in `bot/application/symbol_analyzer.py` and `bot/tracking.py`. | `prepare_symbol()` now runs via `asyncio.to_thread`, and the live tracking path persists feature snapshots via `asyncio.to_thread` under an async lock. |
| Config rejected the `sl_buffer_atr` scale already used by strategy defaults/tests. | Confirmed in `bot/config.py`. | Validation now accepts the ATR-fraction scale (`>= 0.05`), and `config.toml` plus `config.toml.example` were aligned to the strategy-scale defaults. |
| Live ML labeling allowed implicit OHLCV lookahead. | Confirmed in `bot/ml/training_pipeline.py`. | Raw OHLCV fallback labeling is now offline-only and requires an explicit `allow_offline_ohlcv_labels=True` opt-in. Outcome/PnL labels remain the live-safe default. |
| Live ML could silently fall back to baseline models. | Partially confirmed in `bot/ml/filter.py` and `bot/ml/signal_classifier.py`. | Live ML now disables itself when the available classifier resolves to `centroid_baseline` or `linear_baseline`, and model-kind telemetry is carried in the classifier/filter metadata. |
| Backtesting was equity/leverage-oriented instead of signal lifecycle oriented. | Confirmed in prior `bot/backtest/engine.py`. | `VectorizedBacktester` now simulates `pending -> active -> tp1/tp2/sl/expired` outcomes and reports lifecycle metrics while preserving the existing entry point. |
| Runtime notifier path was Telegram-only. | Confirmed in `bot/application/container.py` and `bot/messaging.py`. | Added provider-configured webhook broadcasters for Slack, Discord, and generic webhooks behind `[bot.notifiers]`, while keeping Telegram as the default provider. |
| Runtime SMC detectors diverged from the audit reference implementation and lacked shared zone-state semantics. | Confirmed in `bot/strategies/fvg.py`, `order_block.py`, `liquidity_sweep.py`, `bos_choch.py`, and `breaker_block.py`. | Added a shared Polars-native SMC helper layer derived from `audit/smc_lib`, verified parity against the reference EURUSD fixtures for FVG, swings, BOS/CHoCH, order blocks, and liquidity pools, and migrated the five live strategies to those shared helpers with `fresh` / `mitigated` / `invalidated` zone-state handling. |
| Remaining repo-wide market-regime/contract cleanup from the audit was fully absent. | Mixed. The helper/strategy remediation landed, but broader feature-contract cleanup and scaffold quarantine are still incomplete in this patch set. | Partially deferred. Runtime/config/backtest/ML safety and the shared SMC layer are now in place; feature-contract and scaffold-surface cleanup remain follow-up work. |
